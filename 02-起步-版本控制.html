版本控制系统分类： 本地版本控制系统 集中化版本控制系统 代表：SVN 分布式版本控制系统 代表：git 三种区： 工作区、暂存区、git仓库 三种状态： 已修改modified 已暂存staged 已提交committed git init 初始化仓库 git status 查看文件状态 git status -s 精简地查看文件状态 git add 开始跟踪文件。使用该命令后，文件会处于暂存状态 git commit 提交文件到git仓库中 git commit -m 添加提交信息，对提交的内容进行描述 git checkout -- 当文件在工作区中修改之后，还没有提交git仓库，使用该命令可以撤销工作区中的修改，并且无法恢复 git reset HEAD 文件名 从暂存区中移除指定的文件 git commit -a -m "" 跳过暂存区，直接将工作区中的文件提交git仓库 git rm -f 文件名 同时从git仓库和工作区中移除文件 git rm --cached 文件名 只从git仓库中移除文件，工作区中仍保留 忽略文件： 对于一些不需要加入git管理，但是又不希望一直出现在未跟踪列表中的文件，可以创建.gitignore的配置文件 1、#开头的是注释 2、/结尾是目录 3、/开头防止递归 4、!开头表示取反 5、glob模式进行文件和文件夹的匹配 glob模式：简化之后的正则表达式 查看提交历史： git log 按时间顺序列出所有的提交历史 git log -2 查看最近的2次提交历史 git log -2 --pretty=oneline 在一行上展示提交历史 回退到指定版本： git reset --hard commitID git reflog --pretty=oneline 在旧版本上使用该命令，可以看到所有提交历史，从而跳到指定版本 提交到GitHub上：使用SSH方式 先在本地添加仓库，把本地文件提交git仓库。然后在GitHub上建立仓库，使用SSH将本地仓库推送到GitHub上。 将远程仓库克隆到本地： git clone 远程仓库的地址 分支操作： master主分支用来保存和记录整个项目已完成的功能代码 功能分支是专门用来开发新功能的分支，是临时从master主分支上分叉出来的。当功能开发完毕后，合并到master主分支上 查看分支列表： git branch git branch 新分支名字 创建一个新的分支，但是用户还是处于当前分支，不会切换到新的分支上 git checkout 名字 切换到指定的分支上 git checkout -b 名字 创建并切换到新的分支上 git merge 名字 在master主分支上执行该命令，将指定的分支合并到主分支上
